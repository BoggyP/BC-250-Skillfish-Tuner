# Changelog

## 2.0.0

Major release focused on usability, persistence, monitoring, and BC-250 workflow improvements.

### Added

- Dashboard system information
- Current configuration overview
- Service health monitoring
- CU semicircle gauge
- CU factory topology backup
- Last-good configuration backup and restore
- Persistent CU configuration workflow
- Fan calibration
- Persistent fan settings
- Fan return-to-auto workflow
- Improved overclocking controls
- Voltage safety floor
- System report generation
- Open logs utility
- Smoother telemetry graphs
- Cleaner end-user package structure

### Changed

- Simplified CU management workflow
- Simplified fan control workflow
- Improved service status wording
- Improved dashboard clarity
- Reduced duplicate buttons and confusing actions
- Reworked terminology for better BC-250 clarity

### Removed

- Benchmarking and stability-test subsystem
- vkmark/clpeak integration from the main app
- Benchmark-related legal notices
- Redundant development/test buttons
- Unnecessary generated/cache files

### Notes

This release focuses on making Killfish Tuner more practical as a daily BC-250 management utility.
